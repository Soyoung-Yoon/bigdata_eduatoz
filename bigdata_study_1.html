빅데이터 개요 & 3V·7V

빅데이터 3V / 4V / 5V / 7V

★★★★★
Volume Variety Velocity Value Veracity
핵심 개념 ▼
  • Volume (규모) — 데이터의 크기·양. 예: 구글 자동번역 시스템
  • Variety (다양성) — 데이터 유형: 정형 / 반정형 / 비정형으로 분류
  • Velocity (속도) — 데이터가 생성되고 분석되는 속도
  • Value (가치) — 데이터로부터 얻는 잠재적 가치 또는 비즈니스 인사이트
  • Veracity (진실성) — 데이터의 정확성·신뢰성
  • Validity (유효성)
  • Volatility (휘발성)

데이터 유형: 정형 / 반정형 / 비정형

★★★★★
정형반정형비정형
핵심 분류▼
  • 정형 데이터 — 고정된 스키마, RDBMS, 데이터 웨어하우스, 거래 가격 데이터
  • 반정형 데이터 — 메타 데이터, HTML, XML, JSON(자바스크립트), Web Log, 태그. 파싱을 통해 구조 변환 가능
  • 비정형 데이터 — 정해진 형식 없음, 잠재적 가치 높음. 스트리밍 음악, 블로그, 대화 내용, 텍스트(형태소 단위 분석), 유튜브 영상, 이미지 등

데이터 형태: 정성적 / 정량적

★★★★☆
정성적(질적)정량적(양적)
형태 분류▼

  • 정량적 데이터 — 양적 데이터, 수치/기호/도형, 정형. 계수 데이터는 범주형으로 변환 가능. 예: 기온 예보, 정책 만족도, 통학 시간, 풍속, 강수량

데이터 척도: 명목 / 서열 / 등간 / 비율

★★★★★
명목서열등간비율
핵심 척도▼
  • 명목척도 — 산술 계산 불가. 예: 혈액형, 성별, 출생지, 부서
  • 서열(순위)척도 — 순위 존재, 양적 비교 불가. 예: 학점, 메달, 만족도
  • 등간척도 — 양적 비교(덧셈/뺄셈) 가능, 절대 0점 없음. 예: 온도, 물가, 지능
  • 비율척도 — 절대 0점 존재, 사칙연산 가능, 데이터가 가장 많이 발생. 예: 나이, 키, 불량품 개수

빅데이터 조직 구조

★★★★★
집중형기능 중심분산형
3가지 조직 유형▼
  • 집중형 — 독립적인 분석 전담 조직. 전사 분석 과제 우선순위 정해 추진. 분석 업무 중복 또는 이원화 가능성
  • 기능 중심 — 별도 조직 없음. 전사적 관점에서 핵심 분석이 어려움
  • 분산형 — 분석 조직 인력이 업무 부서에 배치. 분석 결과를 신속하게 실무 적용. 부서 분석 업무와 역할 분담을 명확히 해야 함

하둡(Hadoop) 핵심 구성요소

★★★★☆
HDFSMapReduceKey-Value
하둡 구조▼
  • 핵심 3요소: HDFS(분산 파일 저장), MapReduce(분산 처리), Key-Value
  • 하둡 에코시스템: Zookeeper, Oozie, HBase, Hive, Tajo, Pig, Impala, Mahout, HCatalog, Spark, Avro ...
  • 맵리듀스 디자인 패턴: 필터링, 요약, 조인, 메타, 입출력, 데이터 조직화

🔗 관련 개념

HDFS: GFS 아키텍처 클로닝. 블록 128MB. NameNode 오류 시 읽기/쓰기 불가. 범용 하드웨어 사용

가명정보 vs 익명정보

★★★★★
가명정보익명정보
핵심 구분▼
  • 가명정보 — 다른 정보와 결합하면 복원 가능. 개인정보 보호법 적용받음
  • 익명정보 — 복원 불가능. 개인정보 보호법 적용받지 않음
  • 익명처리 및 가명처리를 하는 것을 고지할 필요 없음
  • 개인정보 파기는 고지할 필요 없음

비식별화 기법 (식별자 제거)

★★★★★
가명처리총계처리마스킹범주화
5가지 기법▼
  • 가명처리 — 다른 값으로 대체
  • 총계처리 — 통계값 적용. 수량이 많아야 함. 정밀한 분석 어려움. 재배열 시 개인 추정 불가
  • 데이터값 삭제 — 해당 값을 삭제(없애는 것)
  • 데이터 범주화 — 대푯값, 구간값으로 변환. 종류: 감추기, 랜덤 라운딩(올림/내림-확률적), 제어 라운딩, 범위 방법
  • 데이터 마스킹 — 대체값, 노이즈(*, 공백)

🔗 관련 개념

차등 정보 보호: 개인정보 + 노이즈 = 개인정보보호 + 데이터 분석 모두 가능 (교란, 섭동)

분석 방법론 (KDD · CRISP-DM · 빅분기 5단계)

빅데이터 분석 방법론 5단계

★★★★★
분석기획데이터 준비데이터 분석시스템 구현
방법론 구조▼
  • 구조: 계층적 프로세스 모델 — 단계, 태스크, 스텝의 3층 구조
  • 5단계: 분석 기획 → 데이터 준비 → 데이터 분석 → 시스템 구현 → 평가 및 전개
  • 분석 기획 태스크: 비즈니스 이해 및 범위 설정(SOW) → 프로젝트 정의 및 계획 수립(WBS) → 프로젝트 위험 계획 수립
  • 위험 대응 방법: 회피 / 전이 / 완화 / 수용
  • 데이터 준비 태스크: 필요 데이터 정의, 데이터 스토어 설계, 데이터 수집 및 정합성 점검(+데이터 정제)
  • 데이터 분석 태스크: 분석용 데이터 준비(유의미한 변수 선택), 텍스트 분석, 탐색적 분석(EDA), 모델링, 모델 평가 및 검증
  • 시스템 구현: 설계 및 구현, 시스템 테스트 및 운영
  • 평가 및 전개: 모델 발전 계획, 프로젝트 평가 및 보고

🔗 관련 개념

EDA: 데이터 구조를 가정하지 않고 데이터의 구조와 특성을 탐색하여 이해하는 과정

KDD 5단계

  • 일관성(Consistency) — 구조·값·표현 형태가 일관되게 정의·일치하는지
  • 유효성(Validity)
