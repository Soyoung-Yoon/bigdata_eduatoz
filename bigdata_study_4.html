회귀 평가지표 종류

★★★★★
MAEMSERMSEMAPER²
회귀 평가 핵심▼
  • 오차 지표: MAE, MSE, RMSE, MSLE, RMSLE, MPE, MAPE → 0에 가까울수록 좋은 성능
  • 설명력 지표: 결정계수(R²), 수정 결정계수(Adjusted R²) → 1에 가까울수록 좋은 성능
MAE (평균 절대 오차)
MAE = Σ|yᵢ - ŷᵢ| / n
MSE (평균 제곱 오차)
MSE = Σ(yᵢ - ŷᵢ)² / n
RMSE (MSE의 제곱근)
RMSE = √MSE
MAPE (평균 절대 백분율 오차)
MAPE = (100%/n) × Σ|yᵢ - ŷᵢ| / yᵢ
MPE (평균 백분율 오차)
MPE = (100%/n) × Σ(yᵢ - ŷᵢ) / yᵢ
결정계수 R²
R² = SSR/SST = 1 - SSE/SST

  • 민감도(Sensitivity) = 재현율(Recall) = TP Rate = TP / (TP+FN)
  • 특이도(Specificity) = TN / (TN+FP)
  • 정밀도(Precision) = TP / (TP+FP)
  • 정확도(Accuracy) = (TP+TN) / (TP+TN+FP+FN)
  • FP Rate = FP / (TN+FP) = 1 - 특이도
  • 오분류율(Error Rate) = (FP+FN) / (TP+TN+FP+FN)
  • FN Rate = FN / (TP+FN)

⚠️ 시험 포인트 — 외우기: "실Sen 예Pre"

재현율(Recall): 실제 양성 중 양성으로 예측한 비율 (분모=실제 양성)
정밀도(Precision): 양성으로 예측한 것 중 실제 양성인 비율 (분모=예측 양성)

F1 Score · Fβ Score

★★★★★
F1 ScoreFβ Score불균형 데이터
F-Score 핵심▼
  • F1 Score: 정밀도와 재현율의 조화평균. 불균형 데이터 상황에서 유용. 두 지표에 동등한 가중치
F1 Score
F1 = 2 × Precision × Recall / (Precision + Recall)
Fβ Score (β에 따라 재현율에 가중치)
Fβ = (1+β²) × Precision×Recall / (β²×Precision + Recall)
  • Fβ Score: β값에 따라 재현율에 더 큰 가중치를 두는 방식. 재현율에 정밀도의 β배만큼 가중치 부여
  • β > 1: 재현율을 더 중요시 / β < 1: 정밀도를 더 중요시 / β = 1: F1 Score

Cutoff에 따른 민감도·특이도 변화

★★★★★
C=0C=0.5C=1
Cutoff 특성▼
  • C = 1: 모든 예측을 음성 클래스로 분류 → 민감도=0, 특이도=1
  • C = 0: 모든 예측을 양성 클래스로 분류 → 민감도=1, 특이도=0
  • C = 0.5: 민감도와 특이도는 데이터의 양성/음성 비율과 모델 성능에 따라 달라짐
  • C가 작아지면 특이도는 감소하고, 민감도는 증가

⚠️ 시험 포인트

교차점이 반드시 C=0.5인 것은 아니며, C=0.5에서 민감도=특이도라고 단정할 수 없음!

ROC 곡선 · AUC

★★★★★
FP Rate(X축)민감도(Y축)AUC
ROC 핵심▼
  • ROC 곡선: X축 = FP Rate(1종 오류 비율), Y축 = 민감도(TP Rate). 이 두 평가 값의 관계로 모형 평가
  • Curve 범위:
    • (민감도, FP-Rate): (0,0)~(1,1)이 시작과 끝
    • (민감도, 특이도): (0,1)~(1,0)이 시작과 끝
  • AUC: ROC 그래프 밑부분의 면적. 1에 가까울수록 좋은 분류 모형
  • Perfect Classifier: 긍정·부정 모두 맞추는 위치. (FP-Rate, 민감도) = (0, 1)

향상도 도표: 랜덤 모형 대비 해당 분류모형의 성과가 얼마나 향상되었는지 등급별로 파악

파라미터 vs 하이퍼파라미터

★★★★★
파라미터(매개변수)하이퍼파라미터(초매개변수)
파라미터 구분▼
  • 파라미터(매개변수): 모델이 데이터로부터 학습을 통해 결정되는 값
    예: 회귀모델의 회귀계수, 인공신경망의 가중치(weight)
  • 하이퍼파라미터(초매개변수): 학습 전에 사람(전문가)이 미리 설정하는 값
    예: k-NN의 K, Ridge/Lasso의 α, 인공신경망의 학습률·배치 크기·은닉 유닛(노드)·은닉층·epoch 수

하이퍼파라미터 튜닝(최적화) 4가지

★★★★★
수동탐색격자탐색무작위탐색베이지안
튜닝 방법 비교▼
  • 수동 탐색: 사용자가 직접 변경해 적용. 비효율적이며 재현성이 낮음
  • 격자 탐색(Grid Search): 하이퍼파라미터값의 목록을 만들고 모든 가능한 조합에 대해 모델 평가. 목록에 없는 값은 적용 불가
  • 무작위 탐색(Random Search): 범위 내에서 랜덤하게 값을 선택. 최적값 보장 없음
  • 베이지안 최적화: 새로운 하이퍼파라미터값 조사 시 이전 경험에 대한 확률을 반영하는 방법

편향-분산 분해(Bias-Variance Decomposition)

★★★★★
편향(과소적합)분산(과대적합)Noise
편향-분산 트레이드오프▼
기대 테스트 오차 분해
Expected Test Error = Bias² + Variance + Noise
  • 편향(Bias) 높음: 예측값의 평균이 실제값과 멀리 떨어짐. 지나치게 단순한 모델. 과소적합(Underfitting) 야기
  • 분산(Variance) 높음: 특정 학습 데이터에 과도하게 적합. 데이터셋 변경 시 예측값도 크게 변함. 지나치게 복잡한 모델. 과대적합(Overfitting) 야기
  • Noise: 데이터 내 포함된 백색잡음. 데이터에 내재된 불확실성. 정상 시계열

📈 학습 오차 vs 검증 오차 곡선

• 학습 오차는 반복 학습에 따라 감소하지만, 검증 오차는 일정 지점 이후 다시 증가 가능
• 두 오차의 차이가 클수록 과대적합 가능성 높음
• 두 오차가 모두 낮을 때 → 일반화 성공
• 과대적합되면 검증 곡선의 성능이 낮아짐

모형 해석 기법▼
  • 해석 가능한 모형(Interpretable Models): 모델 자체가 단순하여 입력변수와 예측 관계를 직관적으로 해석 가능. 예: 선형회귀, 로지스틱회귀, 의사결정나무

★★★★☆
기준 기반비교 기반규준 기반
평가 유형 3가지▼
  • 기준 기반 평가: 사전에 설정된 절대 기준(Criterion)에 따라 목표 수준에 도달했는지 절대 평가(Absolute Evaluation)하는 방식
  • 비교 기반 평가: 기준 모델 또는 경쟁 모델과 성과를 상대적 비교(Relative to Baseline)를 통해 평가

★★★★☆
ROI·NPV·IRR정성적 평가
비즈니스 기여도▼
  • 모델이 실제 조직의 비용, 매출, 효율성, 리스크, 고객 만족 등의 측면에서 얼마나 가치(Value)를 창출했는지 정량·정성적으로 평가

  • 정성적 평가 요소: 비재무적 개선 효과 — 의사결정 품질 향상, 업무 효율성 개선, 리스크 관리 수준 향상, 조직의 데이터 역량 강화, 이해관계자 만족도 증가

  • 분석 모형 전개(Deploy): 분석 모델을 운영 환경과 통합하고 실행하는 것. 개발과 피드백 적용을 반복적으로 수행하여 성능 향상 가능
